## Assessment

Day 2 was useful but too aggressive. The system found real intraday setups and the risk monitor successfully cleaned up several bad carryover problems, but the entry workflow allowed too many positions to accumulate too quickly. The account was down on the latest saved snapshot, and most remaining open positions were slightly negative.

The main operational problem was not a lack of data. It was decision discipline after a signal fired. The system needed stricter controls around duplicate same-symbol entries, active position count, zero-trade backtests, stale losers, and take-profit distance. Static bracket take-profits were still too far from market for a fast day-trading workflow, so winners could fade before the limit filled.

## Changes Applied From Day 2 Lessons

- Zero-trade backtests now fail as insufficient evidence instead of passing.
- Safe and risky take-profit caps have been tightened for intraday behavior.
- Active position caps were added: safe 4, risky 5.
- Stale-loser exits were added to the live monitor.
- Order-planning diagnostics were added to explain why a target did not become an order.
- A same-cycle cross-profile buy guard was added so if safe buys a symbol, risky cannot immediately buy the same symbol in that same cycle before the fill/bracket state has settled.
- Doctrine and runner docs were updated with the new rules.
